# Tryll Meet Notes — план (v2, Vercel-центричный)

**Цель:** трекать календарь → бот сам заходит в созвон в момент старта → транскрипция с именами спикеров → заметки от Claude (Sonnet) → готовый .docx в папку на Google Drive компании.

Архитектура и инструкция по развёртыванию — в [README.md](README.md). Здесь — что осталось сделать и в каком порядке.

## Решения (зафиксировано)

| Вопрос | Решение | Почему |
|---|---|---|
| Где живёт логика | **Vercel** (serverless functions + cron) | Как просил Максим; «мозгу» хватает минутных запусков |
| Где живёт бот | **Vexa** (cloud ~$0.50/час ИЛИ self-hosted на сервере Tryll) | Serverless физически не держит headless-браузер в звонке 60+ мин; Vexa = open source, спикеры из UI Meet (настоящие имена) |
| Состояние | **Upstash Redis** (free, через Vercel Marketplace) | Serverless без диска; нужен dedup событий и очередь |
| Заметки | **Claude Sonnet** (`claude-sonnet-4-6`): путь A — API-ключ; путь B — scheduled-агент Claude Code на подписке через `/api/pending` | Коннекторы claude.ai нельзя вызвать из серверного кода; агент по подписке — легальная замена |
| Формат | **.docx** (библиотека `docx`), имя `<Название> — <YYYY-MM-DD>`, папка = серия события | Как просил Максим |

## Риски

| Риск | Митигировать |
|---|---|
| API Vexa отличается от `src/vexa.ts` | Шаг 1 — сверка по докам развернутой версии; правки локализованы в одном файле |
| Бота не пускают из зала ожидания | Бот-аккаунт в домене tryllengine.com; статус `failed` с понятной ошибкой |
| Vercel Hobby: cron раз в день | cron-job.org каждую минуту (бесплатно) или Vercel Pro |
| Тик дольше лимита функции (длинный транскрипт + Claude) | `maxDuration: 300`; при росте — вынести обработку в отдельный вызов |
| Приватные встречи | тег `[norec]` в названии события |
